Clean tiles
refactors the simple tiles example to more clean tiles

In the previous tutorial we made a one-on-one translations of the content parts to tiles. This generated a lot of very small tiles ,*/title.tile, and lot of folders like

  • Home/Index
  • Home/About
  • Account/ChangePassword
  • Account/Logon
  • ...
We will deal with these changes in this tutorial

Highlights

First we will refactor the Home/Index/title.tile out. We will move the content of this file

Home Page

to the tiles.xml

<?xml version="1.0" encoding="utf-16"?>
<tiles xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xmlns:xsd="http://www.w3.org/2001/XMLSchema">
 <definitions>
  ...

  <tile name="Home.Index" extends="Default.Page.Layout">
  <attribute name="title" type="String" value="Home Page"/>
  <attribute name="body" value="Views.Home.Index.body.tile"/>
  </tile>
  ...

 </definitions>
</tiles>

Having one file in a folder makes no sence. So we can move/rename the fiel Home/Index/body.tile to Home/Index.tile. This results in the following change in tiles.xml

<?xml version="1.0" encoding="utf-16"?>
<tiles xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xmlns:xsd="http://www.w3.org/2001/XMLSchema">
 <definitions>
  ...

  <tile name="Home.Index" extends="Default.Page.Layout">
  <attribute name="title" type="String" value="Home Page"/>
  <attribute name="body" value="Views.Home.Index.tile"/>
  </tile>
  ...

 </definitions>
</tiles>

The rest is more of the same. See the project for end result.
