Actions and urls
handles the difference between<c:url/> and <html:action/>

In the jstl world url's are defined using the <c:url/> this works fine in most situations. The HtmlHelper provides a different strategy with the <html:action/> tags. The big difference is that the last is more suitable and descriptive for controller actions and the first is a more generic solution for loading images, css, javascript and other resources.

The Shared/Logon.tile uses the <html:action/> tag for it's links <c:choose>
 <c:when test="${Request.IsAuthenticated}">
  Welcome <b>${Page.User.Identity.Name}</b>
  [<a href='<html:action Action="LogOff" Controller="Account"/>'>
  Log off</a>]
 </c:when>
 <c:otherwise>
  [<a href="<html:action Action="LogOn" Controller="Account"/>">
  Log on</a>]
 </c:otherwise>
</c:choose>

The next example show how to rewrite a <c:url/> tag to an <html:action/> tag

Shared/Menu.tile <ul id="menu">
  <li>
    <a id="menu_to_index"
        href="<c:url value='~/Home/Index'/>1">Index</a>
  </li>
  <li>
    <a id="menu_to_about"
        href="<c:url value='~/Home/About'/>2">About</a>
  </li>
</ul>

becomes <ul id="menu">
  <li>
    <a id="menu_to_index"
        href="<html:action Action='Index'/>1">Index</a>
  </li>
  <li>
    <a id="menu_to_about"
        href="<html:action Action='About'/>2">About</a>
  </li>
</ul>

Notice that you don't have to define the controller when your action is applied on the last controller used.
Whether you use <c:url/> or <html:action/> is up to you.
