Resources in tiles
Illustrates the use of .resx resources in your tiles

The clean tiles tutorial doesn't do anything for globalization. We will add this to sample project. First we will add a .resx file to the project. And make sure it is loaded on start up. We do this in the global.asax.cs. (There are otherways to load a resource file. Those will be discussed later on in this tutorial). The resource files are loaded through the resource locators. So basicly this means embedded tiles, embedded resources. Tiles as content, resources as content. (In non-embedded the .resx doesn't have to compiled; SharpTiles will do that for you.)

global.asax.cs
protected void Application_Start()
{
  RegisterRoutes(RouteTable.Routes);
  ViewEngines.Engines.Clear();
  var engine = new TilesViewEngine();
  ViewEngines.Engines.Add(engine.Init());
  engine.LoadResourceBundle("ResourceBundles.Views.default");
}

This loads the resource as the default resource bundle. We now can use them in the tiles.

Shared/Layout.tile
...
<div id="footer">
  <p>
    <fmt:message key="default_footer"/>
  </p>
</div>
...

Bundles can also be defined from within tiles. They are cached so don't worry about performance. Defining bundles inline is done as follows:

Shared/Layout.tile
...
<fmt:bundle baseName="ResourceBundles.Views.different"> ...Between these tags, the default bundle now overriden by this bundle. </fmt:bundle> ...

Source location: ResourceBundles project in the Tutorials solution.
